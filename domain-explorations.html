Parked exploration

Reference graphs need a sharper open boundary.

The useful version of this page is not a generic gallery of examples. It should show how Vellis can start from a concrete schema or public standard without pretending that a downloaded ontology is enough to run a regulated workflow.

For now, the supported path is to clone Vellis, run the graph locally, and follow the documented beta walkthrough. Reference graphs will be promoted when builders can inspect and reuse their boundaries confidently.

What a useful release needs

When this expands, it should feel more like installable schema modules: public-standard mappings, narrow workflow examples, and community reference builds with clear boundaries.

01

Public-standard mappings

Turn an existing public ontology, specification, or filing structure into a Vellis-readable schema example.

02

Narrow workflow packs

Show enough of a real workflow to be useful without implying that Vellis alone operationalizes the whole domain.

03

Community reference builds

Let builders share patterns once the repo, example format, and review path are ready.
