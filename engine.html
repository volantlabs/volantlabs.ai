Vellis Engine · first open project

Vellis is the open-source graph engine for agent memory.

Clone the repository and run a local RTG knowledge system for shared agent memory. Vellis ships as a turnkey MCP app and the reusable storage, graph, schema, migration, validation, query, and controller components beneath it.

# macOS prerequisites
› brew install uv just

# clone and set up Vellis
› git clone https://github.com/volantlabs/vellis.git
› cd vellis

› just rtg-eval-info /tmp/vellis-beta-001

Typed graph

Nodes and edges with enforced types — meaning agents can reason with them, not just store them.

Schema language

Declare types, links, and constraints in a readable schema the graph enforces at write time.

Storage

Runs locally with filesystem and SQLite-backed operational state. Your data stays on your machine.

Agent tooling

The repo gives builders the MCP server pattern so Claude, Codex, and other MCP-capable tools can work against graph operations after setup.

Recovery & portability

JSON-safe snapshots, restore paths, and ledger replay keep graph state portable and auditable.

Starter guides

Usage guides, beta prompts, component specs, and a known-good walkthrough live beside the code.

A graph memory engine in five parts

Stable anchors preserve identity, associated facts hold properties, and UUID links carry relationships. Schema and snapshots keep the system valid and portable.
